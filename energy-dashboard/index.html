What this is



A self-contained prototype of a Splunk dashboard for an energy company's IT and OT security operations, built so it can be reviewed without a live Splunk instance. The Simple XML tab contains the matching, pasteable Splunk source — the rendered panels here and the SPL in the XML are aligned one-to-one, though chart values on this page are deterministic illustrative sample data, not live search results.




Before deploying



  
1Replace every index=, sourcetype=, and lookup name with the names actually used in your environment. The ones here are descriptive placeholders.

  
2Confirm OT visibility platform integration (Claroty / Dragos / Nozomi or equivalent) before relying on asset inventory, protocol, or rogue-device panels — coverage and field names vary by vendor and deployment mode.

  
3Build the four reference lookups (ICS-CERT advisories, NERC CIP controls, CIP-004 personnel status, threat intel IOCs) via scheduled searches or scripted inputs before the compliance and threat-intel panels will populate.

  
4Validate field-level access controls: OT asset and ESP-access data is sensitive. Apply role-based access controls (RBAC) in Splunk so this dashboard, and the underlying indexes, are scoped to authorized security and operations staff.

  
5Tune KPI thresholds (the colored range values on single-value panels) to your organization's actual SLAs and risk appetite — the values here are placeholders.





Design notes



Built in Splunk's dark theme with classic Simple XML (not Dashboard Studio) for maximum portability across Splunk Enterprise and Splunk Cloud versions. Panels are grouped into five tabs — Executive overview, IT security, OT/ICS security, NERC CIP compliance, and Threat intel & physical — so an account team can navigate by audience: a CISO or board member typically lives in the first tab, while SOC analysts and OT engineers work the detail tabs. Charts are hand-rolled vanilla SVG with no external chart library or CDN dependency, so the file travels as a single HTML document.




Canadian-market note: if adapting this for a Canadian utility, the CIP compliance tab maps cleanly onto OSFI B-13 (for financially regulated entities with OT exposure) or applicable provincial reliability standards, and Bill C-26/CCSPA reporting obligations can sit alongside the incident board as a parallel compliance lane.
